# Continuation Prompts

These prompts are self-contained and can be used from a fresh clone after
running `python3 scripts/verify_environment.py`.

## Phase 2 Prompt

```text
Implement Phase 2 for the Salsa20-only CTF solution: differential-linear
distinguisher search and verification.

Scope:
- 4-round Salsa20 only.
- Differential-linear cryptanalysis only.
- Do not use the discarded Salsa20 paper as a primary reference.
- Do not implement ChaCha.
- Do not implement Forro.
- Do not claim full key recovery.

Current baseline:
- Phase 1 Salsa20 core is implemented in src/salsa20_4round.py.
- Tests are in tests/test_salsa20.py.
- Validate with: python3 -m unittest discover -s tests
- Validate compilation with: python3 -m py_compile src/*.py scripts/*.py

Tasks:
1. Implement src/masks.py with:
   - mask represented as list of (word_index, bit_index)
   - apply_xor_difference(state, mask)
   - linear_parity(state, mask)
   - mask_to_string(mask)
   - random_single_bit_masks()
   - validate_mask(mask)

2. Implement src/differential_linear_distinguisher.py with two modes:
   - search: try small input difference masks and output linear masks, estimate
     probability, bias, correlation, standard error, and approximate 95 percent
     confidence interval.
   - verify: re-run the best candidate with a larger independent sample count.

3. Use 4-round Salsa20 core mode without feed-forward unless explicitly
   documented otherwise.

4. Add reproducibility controls:
   - fixed seed
   - configurable sample count
   - configurable number of candidate masks
   - command-line arguments

5. Save outputs:
   - results/distinguisher/salsa20_search_results.csv
   - results/distinguisher/salsa20_search_results.json
   - results/distinguisher/salsa20_verified_distinguisher.csv
   - results/distinguisher/salsa20_verified_distinguisher.json

6. Update scripts/run_distinguisher.py so it runs search mode, verifies the
   best candidate, and prints the best distinguisher.

7. Add tests for:
   - mask parity correctness
   - apply difference correctness
   - deterministic search under fixed seed
   - result files generated by a small smoke run

8. Update reports/salsa20_4round_report.md with a Phase 2 section covering:
   - objective
   - method
   - why core mode without feed-forward is used
   - search strategy
   - output files
   - limitations

9. Run:
   python3 -m unittest discover -s tests
   python3 scripts/run_distinguisher.py
   python3 -m py_compile src/*.py scripts/*.py

10. Summarize:
   - best found Delta and Gamma
   - observed correlation and bias
   - result files generated
   - limitations
```

## Phase 3 Prompt

```text
Implement Phase 3 for the Salsa20-only CTF solution: a reduced-subkey
candidate-ranking extension built on the verified 4-round differential-linear
distinguisher.

Scope:
- 4-round Salsa20 only.
- Differential-linear cryptanalysis only.
- Reduced-subkey candidate ranking only.
- Do not claim full 256-bit key recovery.
- Do not implement ChaCha or Forro.

Tasks:
1. Read the Phase 2 result files under results/distinguisher/.
2. Select the verified distinguisher from
   results/distinguisher/salsa20_verified_distinguisher.json.
3. Define a small, explicit reduced-subkey candidate space.
4. Implement candidate scoring in src/key_recovery.py.
5. Update scripts/run_key_recovery.py with command-line arguments.
6. Save CSV and JSON outputs under results/key_recovery/.
7. Add tests for deterministic candidate ranking on a tiny candidate space.
8. Update reports/salsa20_4round_report.md with method, results, and
   limitations.
9. Run:
   python3 -m unittest discover -s tests
   python3 scripts/run_key_recovery.py
   python3 -m py_compile src/*.py scripts/*.py

Summarize the true reduced-subkey rank, candidate-space size, data count,
runtime, and limitations.
```
